Setting Up Vagrant and VirtualBox on Ubuntu

Friday the 14th September, 2018

What is Vagrant?

According the to their website Vagrant is "...a tool for building and managing virtual machine environments in a single workflow.". So what good is that? We can use vagrant to quickly and easily spin up virtual machines to provide clean and consistent development environments. This (hopefully) means we can develop code on a stack that closely resembles the state of the remote server and we can share our environments between colleagues to ensure we are all playing in the same sandpit.

What is VirtualBox?

VirtualBox is a choice from a short list of virtual machine providers that vagrant can use to actually create the virtual machine. VirtualBox is free and if you are developing on a *nix based host machine its the way to go.

Installation

Installing on Ubuntu couldn't be easier. The first step is to install the VM provider VirtualBox: $ sudo apt-get install virtualbox -y
$ virtualbox All being well the VirtualBox GUI should launch. You can use this interface to manage existing VM's, delete stinky ones or force restarts and things like that. Generally though you shouldn't need to use the GUI so you can go ahead and kill the command and the window will close.

The next step is to add Vagrant. Again this is a simple request to the repository: $ sudo apt-get install vagrant -y
$ vagrant -v Vagrant 2.1.4 At the time of writing the latest version of vagrant is `2.1.4`.

Next Steps

You are now setup and ready to go. If you would like to play around with initialising a simple Ubuntu VM then first create a directory to store your vagrant project and cd into it: $ mkdir basic-box && cd basic-box
$ vagrant init ubuntu/xenial64
This will take some time to download the machine image for Ubuntu and spin up the box. Once completed you can run the following commands from within your project directory. * If the VM is running you can SSH into it by simply running: $ vagrant ssh
* To shut it down you can run: $ vagrant halt
* To start it up again run: $ vagrant up
* To make it dead run: $ vagrant destroy

Try to avoid `ctrl+c` while these commands are running or you will end up with a stinky box and will likely have to destroy it via the GUI and recreate it.
